スタンプラリー作成ツールの修正版です。

- `stamp_jpg/not_stamp.jpg` と `stamp_jpg/get.png` への依存を削除
- SVG Data URIを使用したデフォルト画像に置き換え

### 2. **JavaScript構造の修正**

### 3. **動作確認**
- JavaScriptの構文チェック完了
- 全ファイルがエラーなしでパース可能

## 使い方

### 1. 管理者側（スタンプラリーを作る人）

1. `admin.html` をブラウザで開く
2. ラリーのタイトルとコンプリート時メッセージを入力
3. 各スタンプポイントの設定：
   - ポイント名を入力
   - 座標を設定（地図から取得/手動入力/現在地から取得）
   - ヒントを入力（任意）
   - 達成時の画像をアップロード（任意）
4. 「共有URLを生成」ボタンをクリック
5. 表示されたQRコードまたはURLを参加者に共有

### 2. 参加者側（スタンプラリーをやる人）

1. 管理者から受け取ったURLまたはQRコードでアクセス
2. 位置情報の許可を求められたら「許可」を選択
3. 各ポイントに近づくとボタンが有効になる
4. QRコードをスキャン、またはボタンをタップしてスタンプ獲得
5. 全ポイント達成で「Complete!」ボタンが表示される

## 技術仕様

- **位置情報**: Geolocation API使用（GPS精度20m）
- **QRコード生成**: qr-code-styling ライブラリ
- **QRコード読み取り**: html5-qrcode ライブラリ
- **地図表示**: Leaflet.js（管理者ページのみ）
- **データ圧縮**: pako (deflate/inflate)
- **データ保存**: URL圧縮方式 + jsonblob.comフォールバック

## 注意事項

1. **位置情報の精度**
   - GPSの誤差を考慮して、スタンプ取得範囲は20mに設定されています
   - 屋内や高層ビル街では精度が低下する可能性があります

2. **画像サイズ**
   - アップロード画像は自動的に400x400px以下にリサイズされます
   - 大きすぎる画像はURL生成に失敗する可能性があります

3. **ブラウザ対応**
   - モダンブラウザ（Chrome, Firefox, Safari, Edge最新版）推奨
   - カメラアクセスにはHTTPS接続が必要（ローカルテストはOK）

## トラブルシューティング

### 位置情報が取得できない
- ブラウザの位置情報許可を確認
- HTTPSで接続していることを確認（本番環境）

### QRコードがスキャンできない
- カメラの許可を確認
- QRコードを画面中央に配置
- 明るい場所で試す

### 共有URLが長すぎる
- 画像サイズを小さくする
- スタンプポイント数を減らす
- ヒント画像を外部URL参照に変更
